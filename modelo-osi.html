Modelo OSI

Comprende las 7 capas del modelo de referencia para la interconexión de sistemas abiertos

¿Qué es el Modelo OSI?

El modelo OSI (Open Systems Interconnection) es un marco conceptual que estandariza las funciones de un sistema de comunicación en siete capas abstractas.

Fue desarrollado por la Organización Internacional de Normalización (ISO) en 1984 para facilitar la comunicación entre diferentes sistemas informáticos.

Beneficios del Modelo OSI:

  • • Estandarización de protocolos
  • • Facilita el troubleshooting
  • • Modularidad en el diseño
  • • Interoperabilidad entre sistemas

Las 7 Capas del Modelo OSI

7

Capa de Aplicación

Interfaz entre el usuario y la red

+

Función:

Proporciona servicios de red directamente a las aplicaciones del usuario final.

Protocolos:

HTTP/HTTPS FTP SMTP DNS

Ejemplos:

  • • Navegadores web
  • • Clientes de email
  • • Aplicaciones de chat
  • • Transferencia de archivos
6

Capa de Presentación

Traducción, cifrado y compresión

+

Función:

Traduce, cifra y comprime los datos para la capa de aplicación.

Servicios:

Cifrado Compresión Traducción

Ejemplos:

  • • SSL/TLS
  • • JPEG, GIF, PNG
  • • MPEG, MP3
  • • ASCII, EBCDIC
5

Capa de Sesión

Gestión de sesiones y diálogos

+

Función:

Establece, gestiona y termina las sesiones entre aplicaciones.

Protocolos:

NetBIOS RPC SQL

Funciones:

  • • Control de diálogo
  • • Sincronización
  • • Puntos de control
  • • Recuperación de sesión
4

Capa de Transporte

Entrega confiable de datos

+

Función:

Garantiza la entrega completa y correcta de los datos.

Protocolos:

TCP UDP SPX

Servicios:

  • • Control de flujo
  • • Control de errores
  • • Segmentación
  • • Multiplexación
3

Capa de Red

Enrutamiento y direccionamiento

+

Función:

Determina la mejor ruta para enviar datos a través de la red.

Protocolos:

IP ICMP OSPF BGP

Dispositivos:

  • • Routers
  • • Switches de capa 3
  • • Gateways
  • • Firewalls
2

Capa de Enlace de Datos

Control de acceso al medio

+

Función:

Proporciona transferencia de datos libre de errores entre nodos adyacentes.

Protocolos:

Ethernet Wi-Fi PPP

Dispositivos:

  • • Switches
  • • Bridges
  • • Access Points
  • • Tarjetas de red
1

Capa Física

Transmisión de bits

+

Función:

Define las características eléctricas, mecánicas y funcionales para transmitir bits.

Estándares:

IEEE 802.3 RS-232 USB

Componentes:

  • • Cables de red
  • • Hubs
  • • Repetidores
  • • Conectores
