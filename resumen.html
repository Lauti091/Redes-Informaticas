Introducción a las Redes

¿Qué es una red informática?

Una red informática es un conjunto de dispositivos interconectados que pueden comunicarse entre sí para compartir recursos, información y servicios. Estas conexiones pueden ser físicas (cables) o inalámbricas (Wi-Fi, Bluetooth).

Tipos de redes

Los tipos de redes informáticas se pueden clasificar según su tamaño o alcance geográfico, como PAN (Red de Área Personal), LAN (Red de Área Local), MAN (Red de Área Metropolitana) y WAN (Red de Área Amplia), que van desde una distancia personal hasta una cobertura global. Otra clasificación importante es según su arquitectura (como peer-to-peer sin servidor) o según el tipo de conexión, incluyendo las redes inalámbricas (WLAN) que usan radiofrecuencia y las redes privadas virtuales (VPN) que crean un túnel cifrado sobre una red pública.

LAN (Local Area Network)

Red de área local que conecta dispositivos en un área geográfica pequeña como una oficina o edificio.

PAN (Personal Area Network)

Red de área personal para dispositivos cercanos al usuario (Bluetooth, USB).


WLAN (Wireless LAN)

Red de área local inalámbrica que utiliza tecnología Wi-Fi.


MAN (Metropolitan Area Network)

Red de área metropolitana que cubre una ciudad o área metropolitana.

WAN (Wide Area Network)

Red de área amplia que conecta dispositivos a través de grandes distancias geográficas.

CAN (Campus Area Network)

Interconecta varias redes locales (LAN) en un campus universitario o un área geográfica limitada pero más grande que una LAN.

Dispositivos de Red

Router

Dispositivo que conecta diferentes redes y dirige el tráfico de datos entre ellas. Opera en la capa 3 del modelo OSI y toma decisiones de enrutamiento basadas en direcciones IP.

  • Conecta redes diferentes
  • Determina la mejor ruta para los datos
  • Proporciona seguridad mediante firewall

Switch

Dispositivo que conecta múltiples dispositivos en una red local. Opera en la capa 2 del modelo OSI y utiliza direcciones MAC para enviar datos al dispositivo correcto.

  • Conecta dispositivos en la misma red
  • Crea dominios de colisión separados
  • Aprende direcciones MAC automáticamente

Hub

Dispositivo básico que conecta múltiples dispositivos en una red. Opera en la capa física y simplemente repite las señales a todos los puertos.

  • Dispositivo de capa física
  • Crea un solo dominio de colisión
  • Tecnología obsoleta

Access Point

Dispositivo que permite a dispositivos inalámbricos conectarse a una red cableada. Actúa como puente entre redes inalámbricas y cableadas.

  • Proporciona acceso Wi-Fi
  • Extiende la cobertura de red
  • Puede manejar múltiples conexiones

Adaptadores de Red

El adaptador de red es un componente encargado de dotar a un equipo o dispositivo de conectividad, ya sea por cable o inalámbrica.

  • Permite la conexión a la red
  • Soporta conexión por cable o inalámbrica
  • Se instala en dispositivos finales (PC, servidores)

Firewall

Dispositivo o software que controla el tráfico entre redes según reglas de seguridad, protegiendo la red de accesos no autorizados.

  • Filtra tráfico entrante y saliente
  • Puede ser físico o virtual
  • Incluye funciones de seguridad avanzadas (VPN, IDS/IPS)

Sistemas Operativos de Red (NOS)

Windows Server

Sistema operativo de Microsoft diseñado para servidores empresariales.

  • Active Directory para gestión de usuarios
  • Servicios de archivo e impresión
  • Servicios web (IIS)
  • Integración con productos Microsoft

Linux Server

Distribuciones Linux optimizadas para servidores (Ubuntu Server, CentOS, Red Hat).

  • Código abierto y gratuito
  • Alta estabilidad y seguridad
  • Amplia compatibilidad de hardware
  • Comunidad activa de desarrollo

UNIX Server

Sistema operativo estable y seguro usado en servidores corporativos y universidades.

  • Alta fiabilidad y escalabilidad
  • Gestión avanzada de usuarios y permisos
  • Amplio soporte de software de servidor

MacOS Server

Sistema operativo de Apple para entornos de servidor y redes pequeñas/medianas.

  • Gestión de usuarios y grupos
  • Servicios de archivo e impresión
  • Integración con dispositivos Apple

Novell NetWare

Sistema operativo de red histórico, usado principalmente para compartir archivos y recursos en LANs.

  • Gestión de archivos y usuarios
  • Alta velocidad en redes locales
  • Seguridad integrada en la red

Windows Essentials

Versiones simplificadas de Windows Server orientadas a pequeñas empresas.

  • Gestión centralizada de usuarios y recursos
  • Servicios de correo y archivo integrados
  • Facilidad de instalación y administración
  • Compatible con entornos Windows existentes

Funciones principales de un NOS

  • Gestión de usuarios: Creación, modificación y eliminación de cuentas
  • Control de permisos: Definir qué usuarios pueden acceder a qué recursos
  • Servicios de impresión: Gestionar impresoras compartidas en la red
  • Servicios de archivos: Compartir y gestionar archivos en la red

Protocolos de Red

TCP/IP

Conjunto de protocolos fundamentales para Internet. TCP garantiza la entrega confiable de datos, mientras que IP se encarga del direccionamiento.

  • TCP: Protocolo de control de transmisión
  • IP: Protocolo de Internet
  • Base de Internet y redes modernas

HTTP / HTTPS

Protocolos para transferencia de páginas web. HTTPS utiliza TLS para cifrado y seguridad de los datos transmitidos.

  • HTTP: Puerto 80
  • HTTPS: Puerto 443 (cifrado mediante TLS)
  • Base de la World Wide Web

FTP

Protocolo de transferencia de archivos que permite subir y descargar archivos entre sistemas.

  • Puerto 21 para control
  • Puerto 20 para datos
  • Modos activo y pasivo

DHCP

Protocolo que asigna automáticamente direcciones IP y configuración de red a dispositivos.

  • Asignación automática de IP
  • Configuración de gateway y DNS
  • Gestión centralizada de direcciones

DNS

Sistema de nombres de dominio que traduce nombres de dominio a direcciones IP.

  • Resolución de nombres
  • Jerarquía de servidores
  • Caché para mejorar rendimiento

RIP

Protocolo de enrutamiento utilizado por routers para determinar la mejor ruta dentro de una red interna.

  • Routing Information Protocol
  • Usado en redes pequeñas y medianas
  • Actualiza rutas periódicamente

Direccionamiento IP

IPv4

Protocolo de Internet versión 4, utiliza direcciones de 32 bits (4 octetos).

  • Formato: 192.168.1.1
  • Aproximadamente 4.3 mil millones de direcciones
  • Clases A, B, C, D, E

IPv6

Protocolo de Internet versión 6, utiliza direcciones de 128 bits.

  • Formato: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Prácticamente direcciones ilimitadas
  • Mejor seguridad y eficiencia

Máscaras de subred

Definen qué parte de una dirección IP corresponde a la red y qué parte al host.

  • Formato: 255.255.255.0 (/24)
  • Permiten crear subredes
  • Optimizan el uso de direcciones IP

Direcciones públicas y privadas

Públicas: Únicas en Internet, asignadas por ISP

Privadas: Para uso interno, no enrutables en Internet

  • 10.0.0.0/8 (Clase A)
  • 172.16.0.0/12 (Clase B)
  • 192.168.0.0/16 (Clase C)

CIDR

Direccionamiento sin clases que permite asignar bloques de direcciones más flexibles.

  • Formato: 192.168.1.0/24
  • Optimiza el uso de direcciones IP
  • Reduce tamaño de tablas de enrutamiento

Broadcast y Multicast

Tipos especiales de direccionamiento para enviar mensajes a múltiples hosts.

  • Broadcast IPv4: 192.168.1.255
  • Multicast IPv4: 224.0.0.0 a 239.255.255.255
  • Multicast IPv6: ff00::/8

Loopback y Enlace Local

Direcciones especiales para pruebas y comunicación interna en la red.

  • Loopback: 127.0.0.1 (IPv4), ::1 (IPv6)
  • Enlace local: 169.254.x.x (IPv4), fe80::/10 (IPv6)
  • Usadas para pruebas y comunicación sin DHCP

Seguridad en Redes

Firewalls

Sistemas que controlan el tráfico de red basándose en reglas de seguridad predefinidas.

  • Filtrado de paquetes
  • Control de aplicaciones
  • Prevención de intrusiones
  • Registro de actividad

VPN (Virtual Private Network)

Tecnología que crea una conexión segura y cifrada sobre una red pública.

  • Cifrado de datos
  • Acceso remoto seguro
  • Anonimato y privacidad
  • Conexión sitio a sitio

Cifrado

Proceso de codificar información para que solo personas autorizadas puedan acceder a ella.

  • Cifrado simétrico (AES)
  • Cifrado asimétrico (RSA)
  • Funciones hash (SHA)
  • Certificados digitales

Políticas de seguridad

Conjunto de reglas y procedimientos para proteger los recursos de red.

  • Gestión de contraseñas
  • Control de acceso
  • Actualizaciones de seguridad
  • Respaldo y recuperación

IDS / IPS

Sistemas que detectan o previenen intrusiones y ataques en la red.

  • Monitoreo de tráfico en tiempo real
  • Alertas de actividad sospechosa
  • Prevención automática de ataques (IPS)
  • Integración con firewalls

Antivirus / Antimalware

Software que protege los dispositivos frente a virus, malware y ransomware.

  • Detección de virus y troyanos
  • Escaneo en tiempo real
  • Actualizaciones automáticas de firmas
  • Protección en dispositivos conectados a la red

Segmentación de red

Separación de la red en segmentos o VLANs para mejorar seguridad y rendimiento.

  • Aisla tráfico sensible
  • Reduce el impacto de ataques
  • Optimiza ancho de banda
  • Facilita la administración de políticas de seguridad

Topologías de Red

Estrella

Todos los dispositivos se conectan a un punto central (switch o hub).

  • Fácil instalación y mantenimiento
  • Fallo de un nodo no afecta la red
  • Punto central es crítico

Bus

Todos los dispositivos comparten un cable común (backbone).

  • Económica y simple
  • Fácil agregar dispositivos
  • Fallo del cable afecta toda la red

Anillo

Los dispositivos se conectan formando un círculo cerrado.

  • Datos viajan en una dirección
  • No hay colisiones
  • Fallo de un nodo puede afectar la red

Malla

Cada dispositivo se conecta directamente con varios otros dispositivos.

  • Alta redundancia y confiabilidad
  • Múltiples rutas de comunicación
  • Costosa y compleja de implementar

Árbol (Jerárquica)

Dispositivos conectados en forma de jerarquía, combinando varias topologías estrella.

  • Escalable para grandes redes
  • Falla de un nodo afecta solo su rama
  • Administración estructurada y centralizada

Híbrida

Combinación de dos o más topologías para adaptarse a necesidades específicas.

  • Flexibilidad en diseño
  • Permite aprovechar ventajas de varias topologías
  • Más compleja de administrar

Árbol (Jerárquica)

Es una combinación de varias topologías estrella conectadas de manera jerárquica, como un árbol con ramas.

  • Escalable y organizada
  • Fácil de detectar fallos en ramas específicas
  • El fallo en la raíz afecta a toda la red

Híbrida

Es la combinación de dos o más topologías de red diferentes para aprovechar las ventajas de cada una.

  • Flexible y adaptable
  • Permite aprovechar lo mejor de cada topología
  • Puede ser compleja y costosa de implementar

Doble Anillo

Se utilizan dos anillos en paralelo para mejorar la redundancia y la seguridad en la transmisión de datos.

  • Alta tolerancia a fallos
  • Mayor velocidad y confiabilidad
  • Más costosa y compleja que el anillo simple

Punto a Punto (P2P)

Conexión directa entre dos dispositivos sin necesidad de un nodo central.

  • Muy simple y eficiente entre dos equipos
  • No requiere dispositivos intermedios
  • No es escalable para más de dos nodos

Celular

Se utiliza en redes móviles. El área de cobertura se divide en celdas, cada una con una antena base.

  • Permite cobertura amplia dividiendo áreas geográficas
  • Escalable mediante más antenas
  • Costosa de implementar y mantener

Administración de Redes

Herramientas de monitorización

PING

Verifica la conectividad entre dispositivos enviando paquetes ICMP.

Traceroute

Muestra la ruta que siguen los paquetes hasta su destino.

Netstat

Muestra conexiones de red activas y puertos en uso.

Ipconfig / Ifconfig

Muestra y configura parámetros de red del sistema.

ARP

Muestra y gestiona la tabla de direcciones IP ↔ MAC de la red.

NSLOOKUP / DIG

Consultan servidores DNS para obtener la resolución de nombres de dominio.

WHOIS

Devuelve información sobre dominios, direcciones IP y propietarios.

Telnet / Netcat

Permiten probar conectividad con puertos específicos en dispositivos remotos.

CURL / WGET

Herramientas para probar conexiones HTTP/HTTPS y transferir archivos.

MTR (My Traceroute)

Combina ping y traceroute mostrando latencia y pérdida de paquetes en tiempo real.

ROUTE / IP ROUTE

Muestra o configura la tabla de enrutamiento de red.

ETHTOOL (Linux)

Consulta y configura parámetros de interfaces de red (velocidad, duplex, etc.).

Nagios / Zabbix

Plataformas de monitoreo que permiten supervisar redes, servidores y aplicaciones.

  • Alertas en tiempo real ante fallos
  • Paneles de control gráficos
  • Extensible mediante plugins

PRTG Network Monitor

Herramienta de monitoreo de red que mide disponibilidad, tráfico y rendimiento.

  • Supervisión de ancho de banda
  • Detección de cuellos de botella
  • Alertas por correo y notificaciones

Iperf

Herramienta de línea de comandos para medir el ancho de banda y la calidad de la conexión.

  • Pruebas TCP y UDP
  • Medición de latencia y jitter
  • Ideal para diagnósticos de red

Tcpdump

Analizador de paquetes en consola, útil para administradores de sistemas.

  • Captura de tráfico en tiempo real
  • Compatible con filtros avanzados
  • Ligero y rápido

Cacti / MRTG

Herramientas gráficas para monitorear el uso de ancho de banda y otros recursos.

  • Generación de gráficos históricos
  • Basadas en SNMP
  • Visualización sencilla del tráfico

NetFlow / sFlow

Tecnologías usadas en routers y switches para analizar flujos de tráfico.

  • Monitoreo del comportamiento de red
  • Detección de patrones de tráfico
  • Soporte en dispositivos Cisco y otros fabricantes

Wireshark

Analizador de protocolos de red que captura y examina el tráfico de datos.

  • Captura de paquetes en tiempo real
  • Análisis detallado de protocolos
  • Filtros avanzados
  • Interfaz gráfica intuitiva

Nmap

Herramienta de exploración de red y auditoría de seguridad.

  • Descubrimiento de hosts
  • Escaneo de puertos
  • Detección de servicios
  • Identificación de sistemas operativos

SNMP

Protocolo de gestión de red que permite monitorear y configurar dispositivos de manera remota.

  • Recopilación de métricas y estado de dispositivos
  • Alertas automáticas ante fallas
  • Integración con sistemas NMS (Network Management System)

SSH / Telnet

Permite administrar dispositivos de red de forma remota mediante línea de comandos.

  • SSH: cifrado seguro
  • Telnet: sin cifrado (solo en entornos seguros o pruebas)
  • Administración de routers, switches y servidores

VLAN y Segmentación

División de la red en segmentos lógicos para mejorar rendimiento y seguridad.

  • Separación de tráfico por departamentos o servicios
  • Facilita políticas de seguridad
  • Reduce congestión y colisiones

Mejores prácticas de administración

  • Documentación: Mantener diagramas, configuraciones y políticas actualizadas.
  • Monitoreo continuo: Supervisar rendimiento, disponibilidad y tráfico de la red.
  • Respaldos regulares: Copias de seguridad de configuraciones y datos críticos.
  • Actualizaciones: Mantener firmware, sistemas operativos y software de red al día.
  • Seguridad: Implementar políticas, controles de acceso y cifrado de datos.
  • Gestión de usuarios y permisos: Definir roles claros y limitar accesos según necesidad.
  • Automatización de tareas: Uso de scripts o herramientas para mantenimiento, reportes y actualizaciones.
  • Pruebas de rendimiento y carga: Evaluar la eficiencia de la red antes de implementar cambios.
  • Plan de recuperación ante desastres (DRP): Estrategias para restaurar servicios críticos ante fallas graves.
  • Registro y auditoría de eventos: Mantener logs para detectar problemas o intrusiones.
